## 摘要
智能体 AI 需要持久化存储来保存超出大语言模型（LLM）有限上下文窗口的用户特定历史记录。现有的存储系统使用稠密向量数据库或知识图谱遍历（或两者混合），导致高检索延迟和较差的存储可扩展性。我们推出了 Hippocampus，这是一种智能体存储管理系统，它使用紧凑的二进制签名进行语义搜索，并使用无损 Token-ID 流进行精确的内容重构。其核心是动态小波矩阵（DWM），它对这两种流进行压缩和协同索引，以支持在压缩域中的超快速搜索，从而避免了昂贵的稠密向量或图计算。该设计随存储容量线性扩展，使其适用于长周期的智能体部署。实验评估表明，Hippocampus 将端到端检索延迟降低了高达 31 倍，并将单次查询的 Token 占用减少了高达 14 倍，同时在 LoCoMo 和 LongMemEval 基准测试中保持了准确性。

## Abstract
Agentic AI require persistent memory to store user-specific histories beyond the limited context window of LLMs. Existing memory systems use dense vector databases or knowledge-graph traversal (or hybrid), incurring high retrieval latency and poor storage scalability. We introduce Hippocampus, an agentic memory management system that uses compact binary signatures for semantic search and lossless token-ID streams for exact content reconstruction. Its core is a Dynamic Wavelet Matrix (DWM) that compresses and co-indexes both streams to support ultra-fast search in the compressed domain, thus avoiding costly dense-vector or graph computations. This design scales linearly with memory size, making it suitable for long-horizon agentic deployments. Empirically, our evaluation shows that Hippocampus reduces end-to-end retrieval latency by up to 31$\times$ and cuts per-query token footprint by up to 14$\times$, while maintaining accuracy on both LoCoMo and LongMemEval benchmarks.

### 1. 核心问题与整体含义（研究动机和背景）
智能体 AI（Agentic AI）在处理长程任务时，需要持久化内存来存储超出大语言模型（LLM）上下文窗口的历史信息。现有的内存系统主要依赖**稠密向量数据库（RAG）**或**知识图谱（KG）**。
*   **痛点：** 向量搜索涉及高维浮点运算，延迟高；知识图谱遍历复杂，扩展性差；且两者在存储新记忆时都需要昂贵的嵌入（Embedding）生成或总结过程。
*   **目标：** 论文提出了 **Hippocampus**，旨在通过一种全新的、基于压缩域处理的内存基座，解决智能体在长周期部署中的检索延迟高、存储开销大和 Token 消耗过多的问题。

### 2. 方法论
Hippocampus 摒弃了传统的“Token-向量”模式，采用了**双重表示策略**和**动态小波矩阵（DWM）**：
*   **核心思想：** 将内存内容存储为无损的 **Token-ID 流**（用于精确重建）和并行的**二进制签名（Binary Signatures）**（用于语义搜索），并利用 DWM 进行协同索引。
*   **关键技术细节：**
    *   **动态小波矩阵（DWM）：** 扩展了静态小波矩阵，支持在压缩状态下进行增量更新（Append-only）。它提供 `access`（检索）、`rank`（计数）和 `select`（定位）三种原语，复杂度仅为对数级。
    *   **语义哈希与随机索引（Random Indexing）：** 通过轻量级的随机投影技术，将高维语义向量转换为紧凑的二进制签名。相似的语义会被映射为汉明距离（Hamming Distance）较小的签名。
    *   **检索流程：** 
        1.  LLM 从查询中提取关键词。
        2.  关键词转化为二进制签名。
        3.  在 **Signature DWM** 中进行超快速的**汉明球搜索（Hamming-ball Search）**（利用 CPU 原生的位运算 `XOR` 和 `POPCOUNT`）。
        4.  根据检索到的元数据索引，从 **Content DWM** 中无损重建原始 Token 序列。

### 7. 优点
*   **架构创新：** 首次将动态小波矩阵引入智能体内存管理，实现了压缩域内的直接搜索。
*   **极致效率：** 利用位运算代替浮点运算，极大地释放了计算压力。
*   **端到端优化：** 不仅优化了检索（读），还通过免嵌入技术优化了存储（写）。

### 8. 不足与局限
*   **语义粒度风险：** 二进制签名虽然高效，但在处理极度细微或复杂的语义差异时，可能不如全维度的稠密向量精确（尽管实验显示在现有基准上表现良好）。
