## 摘要
大语言模型（LLM）智能体通常根据完整的动作-观察历史来决定动作，这会引入与任务无关的信息，容易导致冗余动作和更高的推理成本。我们提出了进度感知信念更新（PABU），这是一种信念状态框架，通过显式建模任务进度并选择性地保留过去的动作和观察，来紧凑地表示智能体的状态。在每一步中，智能体预测自上一轮以来的相对进度，并决定是否应存储新遇到的交互，仅根据保留的子集来做出未来的决策。在 AgentGym 基准测试的八个环境中，使用相同的训练轨迹，PABU 实现了 81.0% 的任务完成率，比之前采用全历史信念的最先进（SoTA）模型高出 23.9%。此外，PABU 面向进度的动作选择提高了效率，将平均交互步数减少到 9.5 步，相当于减少了 26.9%。消融实验表明，显式进度预测和选择性保留对于鲁棒的信念学习和性能提升都是必要的。

## Abstract
Large Language Model (LLM) agents commonly condition actions on full action-observation histories, which introduce task-irrelevant information that easily leads to redundant actions and higher inference cost. We propose Progress-Aware Belief Update (PABU), a belief-state framework that compactly represents an agent's state by explicitly modeling task progress and selectively retaining past actions and observations. At each step, the agent predicts its relative progress since the previous round and decides whether the newly encountered interaction should be stored, conditioning future decisions only on the retained subset. Across eight environments in the AgentGym benchmark, and using identical training trajectories, PABU achieves an 81.0% task completion rate, outperforming previous State of the art (SoTA) models with full-history belief by 23.9%. Additionally, PABU's progress-oriented action selection improves efficiency, reducing the average number of interaction steps to 9.5, corresponding to a 26.9% reduction. Ablation studies show that both explicit progress prediction and selective retention are necessary for robust belief learning and performance gains.

### 1. 核心问题与研究动机
*   **核心问题**：当前大语言模型（LLM）智能体在处理序列决策任务时，通常依赖于**完整的动作-观察历史（Full History）**。这种方式存在三大弊端：
    1.  **冗余与噪声**：历史记录中包含大量与任务无关的信息，容易导致模型学习到虚假关联。
    2.  **推理成本高**：随着交互步数增加，上下文长度剧增，导致计算开销和延迟大幅上升。
    3.  **决策效率低下**：冗长的上下文容易导致模型产生重复或无效的动作。
*   **研究动机**：借鉴传统强化学习中的“信念状态（Belief State）”概念，探索如何为 LLM 智能体构建一种**紧凑且任务相关**的状态表示，从而在减少计算资源消耗的同时提升决策的准确性。

### 2. 方法论：PABU 框架
PABU（进度感知信念更新）将智能体与环境的交互建模为部分可观测马尔可夫决策过程（POMDP），其核心思想是**显式建模任务进度**并**选择性保留历史信息**。

*   **核心思想**：
    *   **进度感知（Progress-Aware）**：将任务进度作为信念状态的骨干，预测当前动作是否推动了任务完成。
    *   **选择性保留（Selective Retention）**：不再存储全部历史，而是根据学习到的策略，仅保留对未来决策至关重要的观察和动作。
*   **信念状态构成**：
    信念状态 $b_n$ 被定义为一个结构化的文本表示，包含：
    1.  **用户查询 ($q$)**：原始任务目标。
    2.  **当前进度 ($p_n$)**：对任务完成状态的语义抽象。
    3.  **尝试过的动作 ($A_{att}$)**：在当前进度阶段内已执行的动作，防止重复。
    4.  **可用动作 ($A_{available}$)**：从最新观察中提取的合法动作空间。
    5.  **保留的观察 ($O_{saved}$)**：经过筛选的、对后续步骤有用的关键信息。
*   **算法流程与训练**：
    1.  **轨迹增强**：利用教师模型（如 GPT-4）识别成功轨迹中的“关键动作（Critical Actions）”，并合成对应的进度描述和保留标签。
    2.  **训练目标**：采用多任务学习，让 LLM 同时预测：(a) 是否保留当前观察；(b) 更新后的进度；(c) 下一个进度一致的动作。
    3.  **推理流程**：在每一步交互中，模型按“保留决策 $\rightarrow$ 进度预测 $\rightarrow$ 动作选择”的顺序进行单次推理。

### 6. 主要结论与发现
*   **性能大幅提升**：PABU 在 AgentGym 上的平均任务完成率达到 **81.0%**，相比之前的 SoTA（ATLAS）提升了 **23.9%**。
*   **效率显著优化**：平均交互步数从 13.0 步减少到 **9.5 步**（降低 26.9%），且由于上下文更短，推理时的 Token 消耗和时间大幅下降。
*   **跨模型泛化**：PABU 的有效性在不同架构（Llama, Qwen）和不同规模（1B 到 8B+）的模型上均得到了验证。
*   **任务结构决定上下文需求**：实验发现，观察历史对需要跨步骤参考的任务（如 TextCraft）至关重要，而动作历史对动作空间受限的任务（如 Wordle）更关键。

### 7. 优点
*   **紧凑的状态表示**：成功打破了“LLM 智能体必须依赖全历史”的思维定式，通过信念更新机制实现了长程任务的高效处理。
*   **端到端集成**：将信念更新与动作选择集成在同一个 LLM 调用中，无需额外的摘要模型或复杂的检索系统。
*   **可解释性**：显式的进度预测使智能体的决策过程更具透明度，便于开发者调试。

### 8. 不足与局限
*   **离线训练限制**：目前主要基于离线轨迹优化，缺乏在线探索（Online Exploration）能力，在面对从未见过的极端失败案例时可能鲁棒性不足。
*   **进度合成依赖**：训练数据的标注（进度合成、关键动作识别）高度依赖于高性能的教师模型（如 GPT-4），这可能引入教师模型的偏差。
*   **状态抽象的通用性**：虽然在 8 个环境中表现良好，但对于某些极度复杂、无法用简单文本进度描述的任务，其抽象机制可能需要进一步优化。
