## 摘要
大语言模型（LLMs）在长上下文处理中面临重大挑战，包括二次方计算成本、信息遗忘以及检索增强生成（RAG）中固有的上下文碎片化问题。我们提出了一种受认知启发的框架，用于高效的长上下文推理，该框架基于分块压缩和选择性记忆召回，而非处理所有原始标记（tokens）。该框架将长输入分割成块，并使用学习到的压缩器将每个块编码为压缩的记忆表示。门控模块动态选择相关的记忆块，随后由推理模块利用不断演进的工作记忆进行迭代处理，以解决下游任务。压缩器和推理器通过端到端强化学习进行联合优化，而门控模块则作为分类器进行单独训练。实验结果表明，该方法在 RULER-HQA 等多跳推理基准测试中取得了具有竞争力的准确率，将上下文长度从 7K 外推至 175 万个标记，并且与强大的长上下文基线模型相比，在准确率与效率之间取得了良好的平衡。特别是，与 MemAgent 相比，它实现了高达 2 倍的 GPU 显存峰值占用降低和 6 倍的推理加速。

## Abstract
Large Language Models (LLMs) face significant challenges in long-context processing, including quadratic computational costs, information forgetting, and the context fragmentation inherent in retrieval-augmented generation (RAG). We propose a cognitively inspired framework for efficient long-context inference based on chunk-wise compression and selective memory recall, rather than processing all raw tokens. The framework segments long inputs into chunks and encodes each chunk into compressed memory representations using a learned compressor. A gating module dynamically selects relevant memory blocks, which are then iteratively processed by a reasoning module with an evolving working memory to solve downstream tasks. The compressor and reasoner are jointly optimized via end-to-end reinforcement learning, while the gating module is trained separately as a classifier. Experimental results show that the proposed method achieves competitive accuracy on multi-hop reasoning benchmarks such as RULER-HQA, extrapolates context length from 7K to 1.75M tokens, and offers a favorable accuracy-efficiency trade-off compared to strong long-context baselines. In particular, it achieves up to a 2 times reduction in peak GPU memory usage and a 6 times inference speedup over MemAgent.

大语言模型（LLMs）在处理超长上下文时面临三大瓶颈：
*   **计算开销：** 自注意力机制的二次方复杂度导致显存和计算时间随长度激增。
*   **信息遗忘：** 模型难以在海量信息中精准定位关键证据（Lost in the Middle）。
*   **RAG 的局限性：** 传统的检索增强生成（RAG）将文本切碎，破坏了逻辑依赖，难以处理需要跨片段整合信息的“多跳推理”任务。
*   **现有方案不足：** 递归架构（如 MemAgent）推理速度慢，且无法有效压缩信息。

**研究动机：** 受人类记忆机制（长期记忆与工作记忆的分工）启发，提出 **LycheeMemory** 框架，旨在通过“分块压缩+动态召回”实现高效、精准的长文本推理。

### 2. 方法论
LycheeMemory 由三个核心模块组成，通过端到端强化学习（RL）进行协同优化：

*   **核心组件：**
    1.  **压缩器 (Compressor Φcomp)：** 使用 LoRA 增强的基础模型，将原始文本块（如 4096 tokens）压缩为紧凑的 KV-cache 风格表示（Memory Tokens）。
    2.  **门控模块 (Gate Φgate)：** 一个轻量级分类器，根据当前查询（Query）和已有的工作记忆，动态判断某个压缩记忆块是否与当前推理步骤相关。
    3.  **推理器 (Reasoner Φreason)：** 负责维护一个固定长度的“工作记忆”（Working Memory），并根据召回的压缩记忆块迭代更新状态，最终生成答案。

*   **关键技术细节：**
    *   **KV-cache 风格压缩：** 在文本中插入可学习的记忆标记，通过前向传播将语义信息聚合到这些标记的隐藏状态中。
    *   **端到端强化学习：** 采用 **GSPO (Group Sequence Policy Optimization)** 算法。不同于以往孤立优化组件的做法，LycheeMemory 将压缩和推理视为统一策略，让推理结果的奖励信号反向传播回压缩器，使其学会“为了推理而压缩”。
    *   **状态相关检索：** 门控模块不仅看 Query，还参考当前推理的中间状态，从而支持多跳推理中“先找 A，再根据 A 找 B”的逻辑。

*   **认知启发：** 成功模拟了人类“长期存储+短期工作记忆”的模式，解决了长文本处理的结构化问题。

*   **动态门控：** 引入状态相关的门控机制，解决了传统 RAG 难以处理的条件依赖检索问题。
*   **推理效率：** 线性复杂度的推理曲线，使其在处理超长文本时具有极高的实用价值。

*   **单向依赖风险：** 采用单次扫描模式，如果门控模块在早期漏掉了关键信息，模型无法回溯（Backtracking），可能导致推理失败。

*   **基础模型依赖：** 性能高度依赖于底座模型的指令遵循能力。
