## 摘要
用户与基于大语言模型（LLM）的助手之间的长程交互需要有效的记忆管理，然而目前的方法在记忆的训练和评估方面面临挑战。现有的记忆基准测试依赖于静态的离策（off-policy）数据作为上下文，限制了评估的可靠性和可扩展性。为了填补这些空白，我们推出了 AMemGym，这是一个支持针对记忆驱动个性化进行在策（on-policy）评估和优化的交互式环境。AMemGym 采用结构化数据采样来预定义用户画像、状态相关问题以及状态演变轨迹，从而能够低成本地生成高质量且与评估目标一致的交互。由 LLM 模拟的用户通过角色扮演暴露潜在状态，同时保持结构化的状态一致性。基于结构化数据的全面指标指导着助手的评估与优化。广泛的实验揭示了现有记忆系统（如 RAG、长上下文 LLM 和智能体记忆）之间的性能差距及其相应原因。AMemGym 不仅能够实现在竞争方法中的有效选择，还有可能推动记忆管理策略的自我进化。通过将结构化状态演变与自由形式的交互相结合，我们的框架为提升对话智能体的记忆能力提供了一个可扩展且具有丰富诊断信息的环境。

## Abstract
Long-horizon interactions between users and LLM-based assistants necessitate effective memory management, yet current approaches face challenges in training and evaluation of memory. Existing memory benchmarks rely on static, off-policy data as context, limiting evaluation reliability and scalability. To address these gaps, we introduce AMemGym, an interactive environment enabling on-policy evaluation and optimization for memory-driven personalization. AMemGym employs structured data sampling to predefine user profiles, state-dependent questions, and state evolution trajectories, enabling cost-effective generation of high-quality, evaluation-aligned interactions. LLM-simulated users expose latent states through role-play while maintaining structured state consistency. Comprehensive metrics based on structured data guide both assessment and optimization of assistants. Extensive experiments reveal performance gaps in existing memory systems (e.g., RAG, long-context LLMs, and agentic memory) and corresponding reasons. AMemGym not only enables effective selection among competing approaches but also can potentially drive the self-evolution of memory management strategies. By bridging structured state evolution with free-form interactions, our framework provides a scalable, diagnostically rich environment for advancing memory capabilities in conversational agents.

#### 1. 核心问题与研究动机
*   **核心问题**：现有的 LLM 助手记忆基准测试主要依赖**静态、离策（off-policy）**的数据。这意味着助手是在别人生成的对话历史中进行测试，无法反映助手自身的对话选择对后续交互的影响。
*   **研究动机**：
    *   **评估失真**：离策评估无法捕捉助手在真实交互中的表现，可能导致记忆优化方向错误。
    *   **扩展性差**：手动标注长程对话场景成本极高，难以覆盖多样化的个性化需求。
    *   **缺乏诊断性**：现有指标通常只看最终答案对错，无法区分是“没记下来（Write）”、“没找回来（Read）”还是“不会用（Utilization）”的问题。

#### 2. 方法论：AMemGym 框架
AMemGym 的核心思想是**通过结构化数据驱动自由形式的交互**，实现自动化、可扩展且具有诊断性的在线评估。
*   **结构化数据生成（逆向工程策略）**：
    1.  **用户画像采样**：从 Nemotron-Personas 采样 10 万个画像。
    2.  **问题与状态模式采样**：根据画像生成评估问题，并提取回答这些问题所需的“状态变量”（如：用户的工作年限、团队规模、饮食偏好等）。
    3.  **状态演化轨迹**：模拟用户状态随时间的变化（如从“初级”晋升为“中级”），并生成支撑这些变化的“生活事件”叙述。
    4.  **个性化答案生成**：为每个状态组合预生成标准答案，并通过 LLM 反思步骤确保答案的唯一性和准确性。
*   **在策（On-policy）交互**：
    *   **用户模拟器**：使用 LLM 扮演用户，根据预设的状态演化轨迹，在对话中自然地“暴露”潜在状态信息。
    *   **实时评估**：在每个交互周期结束后，向助手提出预设的评估问题。
*   **诊断性指标**：
    *   **写入（Write）失败**：助手未能将暴露的信息存入记忆。
    *   **读取（Read）失败**：信息存了，但在提问时未能检索出来。
    *   **利用（Utilization）失败**：信息检索出来了，但推理错误导致回答错误。

#### 6. 主要结论与发现
*   **在策评估的必要性**：离策评估会产生“重用偏差”，误导记忆系统的配置选择（如 Top-k 的选择在两种模式下结论不同）。
*   **长上下文瓶颈**：即使是支持超长上下文的原生 LLM，在长程交互后期性能也会剧烈下降，甚至接近随机猜测。
*   **智能体记忆的优越性**：经过设计的智能体写入（Agentic Write）系统显著优于原生 LLM 和标准 RAG，因为它能主动筛选和压缩信息。
*   **自我进化潜力**：证明了助手可以通过环境反馈（Complete Feedback）自主优化其记忆管理提示词（Prompt），降低写入失败率。

#### 7. 优点与亮点
*   **交互性**：突破了静态数据集的限制，实现了真正的在策评估。
*   **诊断深度**：通过“写-读-用”三阶段拆解，能够精准定位记忆系统的薄弱环节。
*   **自动化与可扩展**：全自动的数据生成流程使其能够轻松扩展到不同的领域和更长的对话跨度。
*   **闭环优化**：不仅能评估，还能作为训练环境推动智能体记忆策略的自我演化。

#### 8. 不足与局限
*   **模拟器依赖**：评估质量高度依赖于作为“用户模拟器”的 LLM 的角色扮演能力（尽管文中验证了 DeepSeek 作为模拟器也有效）。

*   **模态限制**：目前仅限于文本交互，尚未扩展到多模态记忆（如视觉信息、语音交互）的评估。
*   **评估偏差**：虽然有反思步骤，但“LLM 评判 LLM”仍可能存在潜在的系统性偏差。
