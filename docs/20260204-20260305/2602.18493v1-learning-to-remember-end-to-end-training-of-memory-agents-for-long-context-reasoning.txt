Abstract 

Long-context LLMs and Retrieval-Augmented Generation (RAG) systems process informa-tion passively, deferring state tracking, contra-diction resolution, and evidence aggregation to query time, which becomes brittle under ultra-long streams with frequent updates. We pro-pose the Unified Memory Agent (UMA), an end-to-end reinforcement learning framework that unifies memory operations and question an-swering within a single policy. UMA maintains a dual memory representation: a compact core summary for global context and a structured Memory Bank that supports explicit CRUD (create, update, delete, reorganize) over key-value entries, enabling proactive consolidation during streaming. To evaluate long-horizon memory behavior, we introduce Ledger-QA, a diagnostic benchmark for continuous state tracking where answers are latent values de-rived from accumulated updates rather than lo-cal span retrieval. Across 13 datasets spanning Ledger-QA, Test-Time Learning, and Accu-rate Retrieval, UMA substantially outperforms long-context and RAG baselines on dynamic reasoning and learning tasks while remaining competitive on standard retrieval benchmarks, underscoring the importance of learned, end-to-end memory management. 

1 Introduction 

Context windows of Large Language Models (LLMs) now spanning millions of tokens(Google, 2025; Anthropic, 2025; OpenAI, 2025; Qwen, 2025; Liu et al., 2024), however, long-context models face fundamental challenges beyond win-dow size. First, computational complexity scales quadratically with sequence length in standard at-tention mechanisms(Vaswani et al., 2017), making

Second, empirical studies reveal a con-text rot phenomenon(Hong et al., 2025): as input length increases, model performance degrades sig-nificantly even when all relevant information is present—models struggle to effectively locate and utilize critical details buried within massive con-texts. Retrieval-Augmented Generation (RAG) ad-dresses scalability by retrieving only relevant pas-sages at query time(Xu et al., 2024), but inherits a deeper limitation shared with long-context ap-proaches: passive information processing . Both paradigms treat context as static raw material, de-ferring all reasoning to query time. As illustrated in Figure 1, RAG systems must reprocess raw re-trieved passages for every query. For reasoning-intensive tasks—such as tracking entity states over time, resolving contradictions, or accumulating ev-idence from fragmented observations—this causes both accuracy degradation and computational re-dundancy. 

Memory-augmented agents offer a fundamen-tally different paradigm: proactive reasoning that consolidates information as it arrives rather than reprocessing raw inputs per query. However, exist-ing approaches face critical limitations. Heuristic-based systems like Mem0(Chhikara et al., 2025), Zep(Rasmussen et al., 2025) rely on hand-crafted prompts and memory schemas, lacking task adap-tivity. Decoupled architectures such as Memory-R1(Yan et al., 2025) and Mem-alpha(Wang et al., 2025b) separate memory management from task execution through independent training, prevent-ing end-to-end optimization—credit from task out-comes cannot flow back to shape memory deci-sions. We propose Unified Memory Agent (UMA) , an end-to-end reinforcement learning framework that jointly optimizes memory operations (create, up-date, delete, reorganize) and task execution within a 1

To diagnose proactive reasoning ca-pabilities, we introduce Ledger-QA , a benchmark requiring continuous state tracking where perfor-mance directly reflects the ability to consolidate information proactively rather than reprocess raw history per query. 2 Method 

2.1 Problem Formulation as MDP 

We formulate the long-context reasoning task as a Markov Decision Process (MDP). To handle se-quences exceeding the context window, we de-compose the input into a stream of chunks C =

{c1, . . . , c n}. The MDP tuple (S, A, P, R) is de-fined as follows: 

State Space ( S). At step t, the state st =(Mt, x t, h t) consists of: • Mt = ( mcore , B): The dual-component mem-ory, comprising a high-level summary ( mcore )and a structured key-value bank ( B). • xt: The current input focus. To circumvent con-text window limits, this is dynamically switched between a local text chunk ck (during mainte-nance) and a specific query q (during QA). • ht: The immediate interaction history, which buffers recent observations and is periodically cleared to manage context length. 

Action Space ( A). We unify all operations into a single action space, categorized by the phase of interaction. Crucially, read-only tools are available in both phases to ensure the agent can verify beliefs or query the ledger at any time. Detailed specifica-tions for all tools (inputs/outputs) are provided in Appendix A. • Memory Operations ( Amem ): Used during the chunk processing phase to maintain the state. 

– Add , Update , Delete : CRUD operations to modify the structured bank B.

– Retrieve , List : Read-only operations to check existing keys or values in B before mod-ification. 

– UpdateCore : A special terminal action that commits the current progress to mcore and trig-gers the transition to the next chunk. • QA Operations ( Aqa ): Used during the query phase to synthesize the answer. 

– BM25(query) , Embedding(query) : Hybrid retrieval over raw text chunks C.

– Retrieve(key) , List() : Access to the struc-tured memory bank B.

– Answer(text) : The terminal action to gener-ate the final response. 

Transition Dynamics ( P). The system evolves through a hierarchical process: • Inner Loop: Standard tool actions update B or append to ht without advancing the input stream. • Outer Loop: The UpdateCore action triggers a state transition where xt advances to ck+1 and ht

is reset. • Phase Transition: Upon processing cn, the sys-tem transitions to the QA phase, where xt be-comes a query q sampled from the task distribu-tion P (q|C).This stochastic transition implies that the optimal policy must construct a memory state Mn robust enough to maximize the expected return over all potential future queries. 2Figure 2: Overview of UMA. Phase I incrementally maintains a structured Memory Bank and core summary via CRUD over chunks; Phase II answers queries using both structured retrieval from the bank and raw-context retrieval.

Input Representation. 3 The Ledger-QA Benchmark 

Existing long-context benchmarks primarily focus on retrieval-centric tasks, where the answer is ex-plicitly located within a specific text span. How-ever, real-world agents often operate in dynamic en-vironments requiring state tracking —continuously updating a structured belief state based on unstruc-tured interactions. To rigorously evaluate this ca-pability, we introduce Ledger-QA , a synthetic benchmark simulating long-horizon personal fi-nance management. Unlike existing datasets de-rived from static corpora, Ledger-QA is program-matically generated to simulate dynamic state up-dates over long horizons. 

3.1 Dataset Construction 

The construction pipeline consists of three stages, designed to mimic the complexity of real-world bookkeeping: 

Long-Horizon Timeline Simulation. We simu-late a user’s consumption behavior over a continu-ous timeline (e.g., a full year). Unlike independent QA samples, the data in Ledger-QA is organized as a chronological stream of N sessions. Each session corresponds to a specific date, ensuring temporal dependencies. 

Natural Dialogue Synthesis. For each session, we employ an advanced LLM (e.g., Gemini-3-Pro) to synthesize natural user-assistant conversations. To ensure realism and complexity, the generation covers a diverse taxonomy of 8 major consump-tion categories (e.g., Dining, Transportation) and incorporates significant noise, such as chit-chat, clarifications, and informal expressions. This de-sign forces the agent to filter irrelevant information and precisely extract structured transaction records (Date, Scene, Amount) from the noisy unstructured text. 

Ground Truth & Query Generation. Parallel to the dialogue, we maintain a structured list of ‘Transaction‘ objects to calculate the ground truth. We generate questions that require reasoning over this structured data rather than simple text match-ing. We design 8 different question categories. The construction methodology and examples are detailed in Appendix B. 

3.2 Why Ledger-QA is Challenging 

Ledger-QA fundamentally differs from standard RAG benchmarks by demanding long-horizon state aggregation rather than local information retrieval. Answers (e.g., "Total spending") are la-tent values derived from summing scattered transac-tions ( v1 +· · · +vn) across the entire timeline. This requires the agent to maintain a continuous, high-fidelity belief state, where a single extraction error at any point propagates to the final result, render-ing simple retrieval or approximate summarization ineffective. 

4 Experiment 

4.1 Experimental Setup 4.1.1 Evaluation Benchmarks 

To rigorously evaluate the Unified Memory Agent across the full spectrum of memory capabili-ties—from dynamic updates to static retrieval—we conduct experiments on three distinct categories of tasks. 

Dynamic State Tracking. Existing benchmarks largely focus on read-only retrieval. To evaluate the capability of maintaining and updating a persistent state over long horizons, we utilize our proposed 

Ledger-QA (described in Section 3). This dataset serves as a litmus test for the agent’s ability to perform precise CRUD operations and maintain a consistent ledger (e.g., calculating running totals) among a stream of noisy dialogue, a capability absent in standard RAG benchmarks. 

Test-Time Learning (TTL). Following the pro-tocol in MemoryAgentBench (Hu et al., 2025), we evaluate on six datasets: TREC-Coarse , TREC-Fine , NLU , Clinic , Banking77 , and PubMed-RCT .In these tasks, the input stream consists of labeled examples rather than narrative text. The agent must "learn" classification rules in-context by storing them in memory and subsequently applying this learned knowledge to answer test queries, assess-5ing the agent’s capacity for inductive reasoning via memory construction. 

Accurate Retrieval (AR). To verify that our memory compression and indexing mechanisms preserve fine-grained details, we select seven estab-lished benchmarks: HotpotQA (Yang et al., 2018), 

LoCoMo (Maharana et al., 2024), LongMemEval 

(Wu et al., 2025), MSC (Xu et al., 2022), PearlTQA 

(Du et al., 2024), SQuAD (Rajpurkar et al., 2016), and ConvoMem (Pakhomov et al., 2025). These tasks demand high-precision information extrac-tion from long documents or conversation histories. 

Evaluation Metric. Given the open-ended nature of agentic interactions, we employ an LLM-as-a-Judge paradigm for robust evaluation. Specifically, we utilize Qwen/Qwen3-30B-A3B-Instruct-2507 

as the evaluator to assess the correctness of the agent’s answers against the ground truth. The de-tailed evaluation prompts and scoring criteria are provided in Appendix C. Detailed statistics and specific taxonomies for all evaluation datasets are provided in Appendix D. 

4.1.2 Baselines 

We compare UMA against a diverse set of base-lines. First, we evaluate standard approaches: Con-cat , which inputs the full context to test the back-bone’s native capacity, and RAG , which retrieves the top-20 chunks using a hybrid strategy combin-ing dense embeddings and sparse BM25 via Re-ciprocal Rank Fusion (RRF). Second, we include recurrent summarization models: MemAgent (Yu et al., 2025) iteratively updates a running mem-ory state based on the query, while MemAgent-woq performs query-agnostic compression, chal-lenging the agent to identify salient information without guidance. Finally, we compare against ad-vanced agentic frameworks: Mem1 (Zhou et al., 2025) interleaves chunk reading with dynamic re-trieval to maintain an evolving internal state, and 

MemAlpha (Wang et al., 2025b) utilizes an RL-optimized policy to manage a composite memory system (core, episodic, and semantic), represent-ing the current state-of-the-art in trainable memory agents. Implementation details are provided in Ap-pendix E. 

4.1.3 Implementation Details Training Dataset Preparation. To foster a ro-bust policy capable of handling diverse memory op-erations, we construct a composite training dataset from three sources: (1) Multi-Hop Retrieval Data 

derived from HotpotQA (Yu et al., 2025) to sim-ulate retrieval-intensive environments; (2) Gen-eral Long-Context Data adapted from MemAlpha (Wang et al., 2025b); and (3) State-Tracking Data 

from our synthetic Ledger-QA to target dynamic updates. Detailed statistics for these datasets are provided in Appendix F. 

Training Setup. We implement our framework using a customized version of veRL (Sheng et al., 2024). The policy is initialized from 

Qwen/Qwen3-4B-Instruct and optimized via Task-Stratified Group Relative Policy Optimization (GRPO) on a cluster of 32 NVIDIA H200 GPUs. Comprehensive hyperparameters and configuration details are deferred to Appendix G.

4.4 Ablation Studies 

To disentangle the contributions of our unified ar-chitecture and the reinforcement learning paradigm, we conduct a comprehensive ablation analysis as shown in the bottom section of Table 1. 

Impact of Memory Maintenance (Phase I). We first examine the necessity of the sequential mem-ory maintenance phase by comparing the full archi-tecture against the retrieval-only variants. • Without RL: Even using the base model, the agent equipped with memory capabilities ( w/o RL , 61.38%) significantly outperforms the state-less baseline ( w/o RL & Phase I , 45.26%). This suggests that simply providing the agent with a "write" capability allows for better information retention than reactive retrieval alone. • With RL: The gap persists after training. The full UMA achieves 76.46% average accuracy, 7surpassing the trained retrieval-only agent ( w/o Phase I ) at 63.81%. 

Impact of RL Training. Next, we evaluate the effectiveness of our Task-Stratified GRPO by com-paring the base models with their RL-trained coun-terparts. • On Retrieval-Only Agent: RL training boosts the performance of the retrieval-only agent from 45.26% ( w/o RL & Phase I ) to 63.81% ( w/o Phase I ). This demonstrates that our reward mechanism effectively teaches the model to uti-lize retrieval tools more precisely, even without a persistent memory. • On Unified Agent: The gain is equally substan-tial for the full architecture, rising from 61.38% (w/o RL ) to 76.46% ( UMA ). The base model often struggles to manage the memory lifecycle coherently, whereas the RL-trained policy learns to perform timely updates and precise lookups, unlocking the full potential of the memory bank. 

Impact of Stratified Advantage Grouping. We further validate our Task-Stratified GRPO by com-paring it with a Global Group variant that normal-izes advantages across all steps indiscriminately. As shown in Table 1, the Global Group variant underperforms our method (73.58% vs. 76.46%). This confirms that mixing heterogeneous rewards (binary QA scores vs. aggregated memory utility) dilutes the learning signal. Our stratified approach, by normalizing within task-specific groups, ensures fairer credit assignment and more effective policy optimization. 

Two-Stage vs. Unified End-to-End Training. 

We further investigate whether UMA can be learned via a decoupled two-stage pipeline that trains memory maintenance and QA separately (UMA (2 Stage) ). As shown in Table 1, this variant reaches 66.29% average accuracy, which is consis-tently below the jointly-optimized UMA (76.46%). We attribute the gap to a train–test mismatch: opti-mizing Phase I in isolation cannot anticipate what information will be maximally useful under the fi-nal QA policy, while optimizing Phase II cannot retroactively shape upstream memory decisions. In contrast, unified end-to-end RL enables direct credit assignment from QA outcomes back to mem-ory operations, encouraging memory entries that are compact, precise, and future-query-aware. 

5 Related Work 

Internal Memory and Context Compression. 

Several approaches extend context by modifying model internals, such as embedding information in parameters (Wang et al., 2025a; Zhang et al., 2025), compressing KV caches (Qian et al., 2025; Li et al., 2024), or utilizing soft prompts (Burt-sev et al., 2020; Ge et al., 2023). While offering high inference efficiency, these methods require white-box access and suffer from inherent capac-ity bottlenecks compared to external architectures (Wang and Chen, 2025), limiting their utility for ultra-long horizons. 

Prompt-based External Memory. To bypass in-ternal limits, frameworks like MemGPT (Packer et al., 2023) and SCM (Wang et al., 2023) manage external storage via OS-like hierarchies or graph schemas. While compatible with black-box models, they rely heavily on zero-shot instruction follow-ing. This creates a dependency on large proprietary models, as smaller models struggle to reliably exe-cute complex memory updates via prompting alone (Wang et al., 2025b), often leading to suboptimal retrieval in dynamic settings. 

Trainable Memory Agents. Recent work shifts towards training models for autonomous memory management. Approaches include recurrent com-pression via RL (Yu et al., 2025), interleaving re-trieval with reasoning (Zhou et al., 2025), and uti-lizing composite memory structures (Yan et al., 2025; Wang et al., 2025b). Unlike our unified ap-proach, these methods typically decouple memory maintenance from reasoning or rely on transient summaries. Our work advances this domain by unifying memory operations and reasoning into a single policy, enabling more effective cross-session state tracking. 

6 Conclusion 

In this paper, we proposed the Unified Memory Agent, an end-to-end framework that jointly opti-mizes memory acquisition and utilization via rein-forcement learning. By employing a Task-Stratified GRPO strategy, our method effectively aligns the heterogeneous objectives of long-horizon memory maintenance and downstream question answering. To address the lack of benchmarks for dynamic state tracking, we introduced Ledger-QA , a dataset requiring persistent updates to an evolving ledger. Experimental results demonstrate that our agent sig-nificantly outperforms baselines on state-tracking 8tasks while remaining competitive on standard re-trieval benchmarks. Ablation studies confirm the necessity of both the learned policy and the explicit memory update mechanism. Looking forward, our work suggests that robust long-context intelligence demands active, learned state management rather than passive retrieval alone. 

Limitations 

While our Unified Memory Agent demonstrates promising capabilities, several limitations remain. First, our current implementation enforces a hard context window limit (e.g., 16k tokens) via trun-cation. Although the memory bank mitigates in-formation loss, an infinite-horizon setting where the agent must manage memory over millions of tokens without any raw context fallback remains an open challenge. Second, the Ledger-QA bench-mark, while rigorous, is synthetically generated. Real-world bookkeeping or state-tracking scenar-ios may involve more unstructured noise and ambi-guity than our current simulation covers. Finally, our training relies on a specific set of tools (CRUD + Retrieval); extending this framework to support multi-modal memory (e.g., images, audio) or more complex tool-use patterns requires further investi-gation.

B Details of Ledger-QA Construction 

The construction of the Ledger-QA benchmark em-ploys a three-stage pipeline designed to simulate realistic, long-horizon bookkeeping scenarios. Un-like traditional methods that generate structured data first and then condition the dialogue on it, we employ a joint synthesis strategy . We provide the LLM with high-level user intents (e.g., "wants to buy coffee"), and the LLM simultaneously gener-ates both the natural dialogue and the correspond-ing structured transaction records. This ensures that details such as amounts and descriptions are contextually appropriate and naturally woven into the conversation. 

B.1 Consumption Scene Taxonomy 

To simulate a diverse personal finance environment, we define a hierarchical taxonomy of consumption scenes. The dataset covers 8 major categories, each with multiple sub-scenes: • Dining: Fast Food, Restaurant, Coffee, Bub-ble Tea, BBQ, Hot Pot, Snacks, Takeout. • Transportation: Subway, Bus, Taxi, Gas, Parking, Train, Flight. • Shopping: Clothing, Electronics, Daily Ne-cessities, Cosmetics, Books, Groceries, Furni-ture. • Entertainment: Movie, KTV, Gaming, Gym, Travel, Concert, Escape Room. • Utilities: Water & Electricity, Property Fee, Phone Bill, Internet, Gas Bill, Rent. • Medical: Medicine, Doctor Visit, Health Checkup, Dental, Glasses. • Education: Training Course, Books & Mate-rials, Online Course, Exam Registration, Tu-ition. • Other: Transfer, Red Envelope, Donation, Pet, Beauty & Salon. 

B.2 Data Generation Pipeline Stage 1: Timeline & Scenario Sampling. For each sample, we first generate a chronological time-line spanning a full year. We randomly sample N

dates (sessions) from this range. For each session, we stochastically select a subset of consumption scenes (e.g., Dining-Coffee , Transportation-Taxi )to form the user’s "intent" for that day. 

Stage 2: Joint Dialogue & Transaction Synthe-sis. We feed the selected date and scene intents into an advanced LLM (e.g., Gemini-3-Pro). The model is instructed to act as a user-assistant pair and generate a JSON object containing two aligned fields: 1. Dialogue: The natural conversation turns. 2. Transactions: The structured ground truth (Amount, Description, etc.) corresponding to the dialogue. Crucially, we do not pre-determine the amounts. The LLM decides reasonable amounts based on the context (e.g., $5 for coffee vs. $500 for a flight), ensuring semantic consistency. 

Stage 3: Question Generation. We extract the ground-truth transaction ledger generated in Stage 2 and programmatically generate questions across 8 distinct templates. The question types include: 1. Time-Range Scene Amount: "How much was spent on Dining from Jan to Mar?" 2. Time-Range Multi-Scene: "Total spending on Dining and Travel in Q1?" 3. Global Total: "What was the total spending across all records?" 4. Max Scene: "Which category had the highest spending?" 11 Prompt for Joint Synthesis (Simplified)

G Training Configuration 

We implement our training framework based on a customized version of veRL (Sheng et al., 2024), optimized for multi-conversation and multi-turn agent rollouts. The policy model is initialized from 

Qwen/Qwen3-4B-Instruct and optimized using AdamW with a learning rate of 1 × 10 −6. We employ the Task-Stratified Group Relative Policy Optimization (GRPO) algorithm with a group size of G = 16 samples per context and a KL diver-gence penalty coefficient of β = 0 .001 .To balance generalization across diverse tasks with the specific demands of intensive state track-ing, we adopt a dual-track training strategy: • Generalist Agent (for TTL & AR): To ensure robust performance across broad retrieval and reasoning tasks, this model is trained on the combined Multi-Hop and General Long-Context datasets for 2 epochs. • Specialist Agent (for Ledger-QA): Given the unique requirement for continuous numerical up-dates and long-horizon consistency in Ledger-QA, we train a separate model specifically on the Ledger-QA training set. This model is trained for 6 epochs to fully adapt to the state-tracking dynamics, allowing us to probe the architectural upper bound for this specialized domain. All experiments are conducted on a cluster of 32 NVIDIA H200 GPUs (4 nodes × 8 GPUs). To manage memory efficiency during long-horizon rollouts, we utilize vLLM for inference with a ten-sor parallelism size of 1 and set the GPU memory utilization to 0.5. The maximum sequence length for both prompts and responses is set to 8,192 to-kens. The total training time spans approximately 2 days for the Generalist Agent and 0.5 days for the Specialist Agent.
